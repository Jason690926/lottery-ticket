### 1.2 完整獎金結構

| 獎項 | 條件（一區+二區）| 獎金 |
|------|-----------------|------|
| 頭獎 | 6 + 1 | 分潤池 + 累積彩金，**保底 NT$2 億起** |
| 貳獎 | 6 + 0 | 分潤池 |
| 參獎 | 5 + 1 | 固定 NT$150,000 |
| 肆獎 | 5 + 0 | 固定 NT$20,000 |
| 伍獎 | 4 + 1 | 固定 NT$4,000 |
| 陸獎 | 4 + 0 | 固定 NT$800 |
| 柒獎 | 3 + 1 | 固定 NT$400 |
| 捌獎 | 2 + 1 | 固定 NT$200 |
| 玖獎 | 3 + 0 | 固定 NT$100 **（2010-09 才新增）** |
| 普獎 | 1 + 1 或 0 + 1 | 固定 NT$100 |

**分潤池機制：**
- 總獎金 = 當期銷售 × 0.55
- 扣除參～普獎固定金額後：頭獎分 89%、貳獎分 11%
- 頭獎槓龜 → 累積到下期
- 保護條款：若參～普獎總和 > 總獎金 70%，普獎改為「總獎金 70% − 參～玖獎總額」均分

**EV 計算重點：** 對中第二區 1 號（普獎 NT$100）即回本一注成本，所以普獎機率對 EV 影響大。

## 二、方法論決策（grill-me 結論，2026-05-03）

### 2.1 核心信念（被 grill 過、確認的版本）

**信念：** 跨期關聯預測（**D-revised**），即「上期 / 上幾期出現的單號、雙號、三號帶出下期、下下期的機率規律」。
- **不是** A（機械偏差，靜態頻率）
- **不是** C（單期內組合結構）
- **是** 馬可夫鏈思維：$P(\text{下期} | \text{上期}) \neq P(\text{下期})$
- 撲克 +EV 思維：找的是「機率優勢」，不是 100% 命中

### 2.2 證偽門檻（使用者承諾遵守）

**100 期回測：用方法選號 vs 純隨機選號，配對 t-test：**
- p < 0.05 且總報酬顯著高於隨機 → 真有訊號
- p ≥ 0.05 或總報酬無差異 → **承諾放棄此方法**

### 2.3 統計校正

- **FDR (Benjamini-Hochberg)**，不用 Bonferroni（會過嚴卡死探索）
- Phase 1 找候選 transition、Phase 2 由 100 期回測閘門驗證

### 2.4 資料切分（不可違反）

```
1,800 期歷史
├── 訓練集（前 ~1,700 期）：找 transition pattern、建模
└── 測試集（最後 100 期）：完全鎖住，Phase 2 才解封
```

## 三、分析方法 catalogue（待逐項評估）

依複雜度由淺到深排序：

1. **單號頻率統計** — 每號歷史開出次數、是否偏離期望值
2. **奇偶/區段/和值分布** — 6 號組合的整體結構
3. **連號 / 跳號模式** — 是否常出現連續號或固定間隔
4. **熱號 / 冷號滾動** — 短期熱門號是否有續熱效應
5. **共現網路（圖論）** — 哪些號常一起被開出（networkx）
6. **馬可夫鏈** — 上期號碼是否影響下期條件機率
7. **時間序列 / 自相關** — 號碼出現是否有週期性
8. **聚類分析** — 把每期的 6 號當特徵向量，看是否有 cluster
9. **機器學習分類** — 給定歷史，預測「某號下期會中」的機率
10. **深度學習序列模型**（LSTM/Transformer，後期視結果決定）

### Streamlit 選號工具（進行中）

**設計原則（grill-me 確認）：**
- 目標：正向選號——給定上期條件，找歷史上出現率較高的號碼
- 資料範圍：滾動最近 200 期（全部資料，不限 split）
- 多條件邏輯：各條件次數加總（加權合計），不做 AND 交集
- 統計誠實聲明常駐於介面底部

**雙區塊架構：**

```
第一區塊（本期號碼）  →  lag-1 分析  →  預測下一期
第二區塊（上一期號碼）→  lag-2 分析  →  預測同一個下一期
                                      ↓
                           ⚖️ 綜合加權排名（可調比重）
```

**核心函式（`query_engine.py`）：**

```python
zone1_analysis(condition_nums, rolling=200, lag=1)
    # → (DataFrame排名表, total_cond_appearances, T)
    # DataFrame 欄位：排名, 號碼, #xx次數(各條件), 合計次數, 合計頻率%, 近N期頻率%

zone1_dual_combined(block1_nums, block2_nums, rolling=200, w1=0.5, w2=0.5)
    # block1 用 lag=1，block2 用 lag=2
    # 各自標準化至 [0,1] 後依 w1/w2 加總
    # → DataFrame：排名, 號碼, B1次數, B2次數, 綜合分數, B1頻率%, B2頻率%, 近N期頻率%

zone2_analysis(condition_z2, rolling=200)
    # → (DataFrame排名表, n_condition_draws)
```

**UI 結構（`app.py`）：**

```
[Header + 說明]
[第一區塊輸入 | 第二區塊輸入]  ← 並排，各有 Z1 multiselect + Z2 selectbox
[分析 Tabs]
  ├── 🔵 第一區塊：lag-1 頻率表 + 堆疊長條 + 折線圖
  ├── 🟠 第二區塊：lag-2 頻率表 + 堆疊長條 + 折線圖（有說明 lag-2 語意）
  └── ⚖️ 綜合加權：w1 滑桿 + 明細表 + 泡泡散布圖 + 貢獻分解長條
[第二區分析]
  ├── 🔵 B1 Tab：頻率表 + 長條圖
  ├── 🟠 B2 Tab：頻率表 + 長條圖
  └── ⚖️ 合併比較 Tab（兩個都填才出現）：排名合併表 + 並排長條圖
[統計免責聲明]
```
