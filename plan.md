## 一、威力彩規則（官方查證版，2026-05-03）

> 來源：台彩官網 + 政府開放資料平台 + 財政部國庫署，**非維基**

### 1.1 遊戲架構

| 項目 | 內容 |
|------|------|
| 第一區 | 1～38，選 6 號（不重複） |
| 第二區 | 1～8，選 1 號 |
| 一注售價 | **NT$100**（非 NT$50；NT$50 是今彩 539）|
| 包牌 | 可複選多組，受經銷商機台限制 |
| 頭獎中獎機率 | 1 / 22,085,448 |

### 1.2 完整獎金結構

| 獎項 | 條件（一區+二區）| 獎金 |
|------|-----------------|------|
| 頭獎 | 6 + 1 | 分潤池 + 累積彩金，**保底 NT$2 億起** |
| 貳獎 | 6 + 0 | 分潤池 |
| 參獎 | 5 + 1 | 固定 NT$150,000 |
| 肆獎 | 5 + 0 | 固定 NT$20,000 |
| 伍獎 | 4 + 1 | 固定 NT$4,000 |
| 陸獎 | 4 + 0 | 固定 NT$800 |
| 柒獎 | 3 + 1 | 固定 NT$400 |
| 捌獎 | 2 + 1 | 固定 NT$200 |
| 玖獎 | 3 + 0 | 固定 NT$100 **（2010-09 才新增）** |
| 普獎 | 1 + 1 或 0 + 1 | 固定 NT$100 |

**分潤池機制：**
- 總獎金 = 當期銷售 × 0.55
- 扣除參～普獎固定金額後：頭獎分 89%、貳獎分 11%
- 頭獎槓龜 → 累積到下期
- 保護條款：若參～普獎總和 > 總獎金 70%，普獎改為「總獎金 70% − 參～玖獎總額」均分

**EV 計算重點：** 對中第二區 1 號（普獎 NT$100）即回本一注成本，所以普獎機率對 EV 影響大。

### 1.3 開獎機制

| 項目 | 內容 |
|------|------|
| 開獎日 | 每週**一**、週**四** |
| 開獎時間 | 20:30（20:00 投注截止後 30 分） |
| 開獎機 | 美國 **Smartplay Inc.**（WLA 認可、TÜV 認證） |
| 球規格 | 塑膠、無法磁化、重量 2.5 ~ 2.775 g |
| 攪拌秒數 | 未公開 |
| 公正單位 | **2024 起為安永會計師事務所**（之前由獨立公正人士） |
| 保養 | 每月維護 + 每年原廠檢驗 |

**對「跨期關聯」假設的影響（給未來自己看）：**
- 三方把關 + 年度檢驗 → 機台官方等於宣示 i.i.d.
- 跨期關聯（D-revised）若存在，等於證明 i.i.d. 設計失敗
- 探索仍有意義，但要心理準備：找到的弱訊號**很可能是統計噪聲**

## 二、方法論決策（grill-me 結論，2026-05-03）

### 2.1 核心信念（被 grill 過、確認的版本）

**信念：** 跨期關聯預測（**D-revised**），即「上期 / 上幾期出現的單號、雙號、三號帶出下期、下下期的機率規律」。
- **不是** A（機械偏差，靜態頻率）
- **不是** C（單期內組合結構）
- **是** 馬可夫鏈思維：$P(\text{下期} | \text{上期}) \neq P(\text{下期})$
- 撲克 +EV 思維：找的是「機率優勢」，不是 100% 命中

### 2.2 證偽門檻（使用者承諾遵守）

**100 期回測：用方法選號 vs 純隨機選號，配對 t-test：**
- p < 0.05 且總報酬顯著高於隨機 → 真有訊號
- p ≥ 0.05 或總報酬無差異 → **承諾放棄此方法**

### 2.3 統計校正

- **FDR (Benjamini-Hochberg)**，不用 Bonferroni（會過嚴卡死探索）
- Phase 1 找候選 transition、Phase 2 由 100 期回測閘門驗證

### 2.4 資料切分（不可違反）

```
1,800 期歷史
├── 訓練集（前 ~1,700 期）：找 transition pattern、建模
└── 測試集（最後 100 期）：完全鎖住，Phase 2 才解封
```

## 三、分析方法 catalogue（待逐項評估）

依複雜度由淺到深排序：

1. **單號頻率統計** — 每號歷史開出次數、是否偏離期望值
2. **奇偶/區段/和值分布** — 6 號組合的整體結構
3. **連號 / 跳號模式** — 是否常出現連續號或固定間隔
4. **熱號 / 冷號滾動** — 短期熱門號是否有續熱效應
5. **共現網路（圖論）** — 哪些號常一起被開出（networkx）
6. **馬可夫鏈** — 上期號碼是否影響下期條件機率
7. **時間序列 / 自相關** — 號碼出現是否有週期性
8. **聚類分析** — 把每期的 6 號當特徵向量，看是否有 cluster
9. **機器學習分類** — 給定歷史，預測「某號下期會中」的機率
10. **深度學習序列模型**（LSTM/Transformer，後期視結果決定）

## 四、開發路線圖（更新版，依 grill-me 結論）

### Phase 0：環境 + 資料就位
- 建 Python venv（pandas, numpy, scipy, matplotlib, statsmodels, jupyter）
- 下載 D423F CSV（政府開放資料），解析威力彩全歷史
- 入 SQLite（一張 `draws` 表）
- 基本 EDA：每號頻率、第二區頻率、年度趨勢視覺化
- **資料切分：訓練 1,700 期 + 測試 100 期**（測試集即時鎖住，命名 `holdout_100`）

### Phase 1：跨期關聯探索（Order A + B 同步）
- 用訓練集（1,700 期）建 38×38 transition matrix（一區）+ 8×8（二區）
- **Test B**：每 row 卡方檢定 → 找出「給定上期某號，下期分布顯著非均勻」的號
- **Test A**：每 cell 卡方 + FDR (BH) q < 0.05 → 找出顯著 transition
- 視覺化 heatmap（A 用）+ barplot（B 用，挑顯著的 row）
- **若 B 全無訊號 → A 一定也無 → 直接跳 Phase 4 結案**

### Phase 2：100 期 holdout 回測
- 用 Phase 1 找出的候選 transition 建立預測模型
- 對 holdout 100 期逐期模擬：
  - 用 1 ~ (期數-1) 的資料推薦下期 6+1 號
  - 同時隨機選 6+1 號做 baseline
  - 對照真實開獎，依獎金結構算 NT$ 報酬
- 配對 t-test：你的方法 vs 隨機，p < 0.05 才算過關
- 畫資金曲線圖
- **過關 → Phase 3；不過關 → 認賠 + 結案**

### Phase 3：模型精煉（**只在 Phase 2 過關才做**）
- 嘗試擴展到 Order C（pair → 單號）— 若資料夠
- 機器學習分類器（logistic regression / gradient boosting）建 P(某號下期會中)
- 多模型 ensemble + 重新 100 期回測
- 思考組合結構（C 方向）做為「分潤優化」加碼

### Phase 4：結案 OR 擴展大樂透
- 結案：寫詳細報告，記錄所有負結果與假設
- 擴展：套用 Phase 0-3 到大樂透（49 選 6）
